## 출력 요구사항

다음 형식의 Markdown 문서를 작성하세요:

```markdown
# Feature Specification: {issue_title}

**Created**: (오늘 날짜)
**Status**: Draft
**Issue**: #{issue_number}

## User Scenarios & Testing

### User Story 1 - (적절한 제목) (Priority: P1)

(Issue 내용을 User Story 형식으로 구체화)

**Acceptance Scenarios**:

1. **Given** [초기 상태], **When** [사용자 액션], **Then** [예상 결과]
2. **Given** [다른 상태], **When** [액션], **Then** [결과]

---

## Requirements

### Functional Requirements

- **FR-001**: (구체적인 기능 요구사항 1)
- **FR-002**: (구체적인 기능 요구사항 2)
- **FR-003**: (추가 요구사항)

### Non-Functional Requirements (선택)

- **NFR-001**: (성능, 보안 등)

---

## Success Criteria

### Measurable Outcomes

- **SC-001**: (측정 가능한 성공 기준 1)
- **SC-002**: (측정 가능한 성공 기준 2)

---

## Edge Cases (선택)

- Edge case 1
- Edge case 2
```

## 주의사항

- User Story는 **독립적으로 테스트 가능**해야 합니다
- Functional Requirements는 **구체적이고 명확**해야 합니다
- Success Criteria는 **측정 가능**해야 합니다
- 전문적이고 기술적인 용어를 사용하세요
